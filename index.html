Infrastructure Overview

Our organization operates a robust business continuity infrastructure with high availability, rapid recovery capabilities, and strategic modernization planning. Current implementation includes:

  • Dual-layered backup systems with on-site and off-site redundancy ensuring minimal downtime.
  • Disaster recovery protocols using Microsoft Hyper-V with recovery capabilities under 3 hours.

This infrastructure ensures operational continuity, mitigates risks, and positions us for long-term scalability.

Data Protection Implementation

Primary Backup: Disk2VHD

  • Method: Bare-Metal System Imaging (P2V)
  • Output: Virtual Hard Disk (.vhdx)
  • Purpose: Rapid system virtualization for DR
  • Storage: IT PC's hard drive (secure)
  • Schedule: Daily full images

Secondary Backup: Veritas

  • Method: System State & File-Level Backup
  • Output: Proprietary Backup Set
  • Purpose: Redundancy and granular restore
  • Schedule: Incremental hourly, full daily

Veritas Storage: NAS Synology

  • Role: Centralized storage for all Veritas backups
  • Function: On-site high-availability storage
  • Features: Scalable, RAID-protected, easy management
  • Capacity: Multi-TB with expansion options

Offsite Mirroring

  • Method: Real-time mirroring of NAS Synology to secure offsite location
  • Function: Protection against site-wide disasters
  • Benefits: Geographic redundancy, compliance-ready
  • Security: Encrypted transmission and storage

Disaster Recovery Procedures

Step 1: Failover Initiation

  • Trigger: Confirmed physical server outage
  • Process: Copy latest .vhdx to Hyper-V host; fallback to Veritas/NAS/offsite if needed
  • Time: 1–3 hours (data size dependent)
  • Owner: IT Operations Team

Step 2: Service Restoration

  • Platform: Microsoft Hyper-V
  • Process: Launch VM from .vhdx or restore from Veritas/NAS/offsite
  • Time: < 5 minutes post-data transfer
  • Validation: Service health checks

Step 3: Failback Process

  • Timing: Post-business hours to minimize impact
  • Process: Migrate data from VM back to repaired physical hardware
  • Objective: Seamless return to primary infrastructure
  • Testing: Post-failback integrity verification

Detailed Documentation

Our infrastructure consists of two mission-critical servers, each protected by hardware redundancy and multi-tiered backups to ensure operational integrity.

SCCMAIN

  • Functions: Active Directory for user authentication, Sage Pastel for financial management, and Email Storage for communications.
  • Storage: Hardware RAID 1 (IcyDock) providing fault tolerance.
  • Integration: Connected to network services and client workstations.

EYETHU1

  • Functions: Point of Sale (POS) for transactions and Time Manager for employee scheduling.
  • Storage: Hardware RAID 1 (IcyDock) for data protection.
  • Integration: Connected to retail terminals and HR systems.

Primary Backup Method – Disk2VHD

  • Process: Creates full system images of SCCMAIN and EYETHU1, stored as .vhdx files on the IT PC's hard drive, ready for Hyper-V deployment.
  • Recovery Time (SCCMAIN): 2–3 hours data copy + 5 minutes launch in Hyper-V.
  • Recovery Time (EYETHU1): 1 hour data copy + 5 minutes launch in Hyper-V.
  • Failover Process: Quick resumption in Hyper-V; post-hours migration back to physical hardware.
  • Testing Schedule: Quarterly simulations to validate integrity.

Secondary Backup Method – Veritas Backup

  • Function: Additional layer for system and file recovery.
  • Usage: Fallback when Disk2VHD unavailable.
  • Benefits: Granular restores and redundancy.
  • Storage: Backed to NAS Synology with offsite mirroring for ultimate protection.
  • Retention Policy: 30 days on-site, 90 days off-site.
  • Minimal Downtime: Hyper-V failover enables recovery in minutes after data transfer.
  • Redundancy: Multi-tiered (Disk2VHD + Veritas + NAS + Offsite) for comprehensive coverage.
  • Flexibility: Supports virtual and physical restores across environments.
  • Business Continuity: Maintains operations for essential services with RTO < 3 hours and RPO < 1 hour.
  • Risk Mitigation: Addresses hardware failures, data corruption, and site disasters.

Server Upgrade Strategy

  • Current State: Windows Server 2012 R2 – Approaching end-of-support.
  • Migration Path: Transition to Windows Server 2025 using Microsoft tools.
  • Testing: testing offline on hyper v to ensure smooth transition.
  • Benefits: Superior security (e.g., enhanced encryption), performance optimizations, and support until 2035.
  • Implementation: Pilot migration Q4 2025; full deployment Q1 2026.

Our infrastructure provides a resilient framework with immediate protection and strategic upgrade planning.

  • Immediate Actions: DR drill scheduled for Q4 2025.
  • Considerations: Evaluating Azure Backup integration for hybrid cloud resilience.
  • Value Delivery: Reduces potential downtime costs and ensures compliance.
