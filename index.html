Server Infrastructure & Business Continuity Plan

A comprehensive strategy for ensuring resilience, security, and future-readiness of critical systems as of September 20, 2025.

Executive Summary

This document outlines a robust business continuity plan for our server infrastructure, focusing on high availability, rapid recovery, and strategic modernization. Key highlights include:

  • Dual-layered backup systems with on-site and off-site redundancy to minimize downtime.
  • Disaster recovery protocols leveraging Microsoft Hyper-V for failover in under 3 hours.
  • Planned migration to Windows Server 2025 for enhanced security and performance.
  • Comprehensive protection for core services including Active Directory, Sage Pastel, Email, POS, and Time Manager.

This approach ensures operational continuity, mitigates risks, and positions the organization for long-term scalability.

Data Protection Strategy

Primary Backup: Disk2VHD

  • Method: Bare-Metal System Imaging (P2V)
  • Output: Virtual Hard Disk (.vhdx)
  • Purpose: Rapid system virtualization for DR
  • Storage: Securely on IT PC’s hard drive
  • Frequency: Daily full images

Secondary Backup: Veritas

  • Method: System State & File-Level Backup
  • Output: Proprietary Backup Set
  • Purpose: Redundancy and granular restore
  • Frequency: Incremental hourly, full daily

Veritas Storage: NAS Synology

  • Role: Centralized storage for all Veritas backups
  • Purpose: On-site high-availability storage
  • Advantage: Scalable, RAID-protected, easy management
  • Capacity: Multi-TB with expansion options

Offsite Mirroring

  • Method: Real-time mirroring of NAS Synology to secure offsite location
  • Purpose: Protection against site-wide disasters
  • Advantage: Geographic redundancy, compliance-ready
  • Security: Encrypted transmission and storage

Disaster Recovery Protocol (Hardware Failure Scenario)

Step 1: Failover Initiation

  • Trigger: Confirmed physical server outage
  • Action: Copy latest .vhdx to Hyper-V host; fallback to Veritas/NAS/offsite if needed
  • Time Est: 1–3 hours (data size dependent)
  • Responsible: IT Operations Team

Step 2: Service Restoration

  • Platform: Microsoft Hyper-V
  • Action: Launch VM from .vhdx or restore from Veritas/NAS/offsite
  • Time Est: < 5 minutes post-data transfer
  • Validation: Service health checks

Step 3: Failback Procedure

  • Timing: Post-business hours to minimize impact
  • Action: Migrate data from VM back to repaired physical hardware
  • Goal: Seamless return to primary infrastructure
  • Testing: Post-failback integrity verification

Detailed Proposal Appendix

Our current setup comprises two mission-critical servers, each protected by hardware redundancy and multi-tiered backups to ensure operational integrity.

SCCMAIN

  • Functions: Active Directory for user authentication, Sage Pastel for financial management, and Email Storage for communications.
  • Storage: Hardware RAID 1 (IcyDock) for fault tolerance.
  • Dependencies: Integrates with network services and client workstations.

EYETHU1

  • Functions: Point of Sale (POS) for transactions and Time Manager for employee scheduling.
  • Storage: Hardware RAID 1 (IcyDock) for data protection.
  • Dependencies: Connected to retail terminals and HR systems.

Primary Backup Method – Disk2VHD

  • Process: Creates full system images of SCCMAIN and EYETHU1, stored as .vhdx files on the IT PC’s hard drive, ready for Hyper-V deployment.
  • Recovery Time (SCCMAIN): 2–3 hours data copy + 5 minutes launch in Hyper-V.
  • Recovery Time (EYETHU1): 1 hour data copy + 5 minutes launch in Hyper-V.
  • Failover Plan: Quick resumption in Hyper-V; post-hours migration back to physical hardware.
  • Testing: Quarterly simulations to validate integrity.

Secondary Backup Method – Veritas Backup

  • Role: Additional layer for system and file recovery.
  • Usage: Fallback if Disk2VHD unavailable.
  • Advantage: Granular restores and redundancy.
  • Storage: Backed to NAS Synology with offsite mirroring for ultimate protection.
  • Retention Policy: 30 days on-site, 90 days off-site.
  • Minimal Downtime: Hyper-V failover enables recovery in minutes after data transfer.
  • Redundancy: Multi-tiered (Disk2VHD + Veritas + NAS + Offsite) for comprehensive coverage.
  • Flexibility: Supports virtual and physical restores across environments.
  • Business Continuity: Maintains operations for essential services with RTO < 3 hours and RPO < 1 hour.
  • Risk Mitigation: Addresses hardware failures, data corruption, and site disasters.

Server Upgrade

  • Current State: Windows Server 2012 R2 – Approaching end-of-support.
  • Planned Path: Migration to Windows Server 2025 using Microsoft tools.
  • Testing: Ongoing in lab environment; full rollout post-validation.
  • Benefits: Superior security (e.g., enhanced encryption), performance optimizations, and support until 2035.
  • Next Steps: Pilot migration Q4 2025; full deployment Q1 2026.

This plan provides a resilient framework for our infrastructure, combining immediate protection with strategic upgrades.

  • Immediate Actions: Conduct DR drill in Q4 2025.
  • Recommendations: Explore Azure Backup integration for hybrid cloud resilience.
  • Investment Justification: Reduces potential downtime costs (estimated $50K/hour) and ensures compliance.
