Data Protection Strategy

Primary Backup: Disk2VHD

  • Method: Bare-Metal System Imaging (P2V)
  • Output: Virtual Hard Disk (.vhdx)
  • Purpose: Rapid system virtualization for DR

Secondary Backup: Veritas

  • Method: System State & File-Level Backup
  • Output: Proprietary Backup Set
  • Purpose: Redundancy and granular restore
▼

Disaster Recovery Protocol (Hardware Failure Scenario)

Step 1: Failover Initiation

  • Trigger: Physical server outage confirmed.
  • Action: Copy latest .vhdx to Hyper-V host.
  • Time Est: 1–3 hours

Step 2: Service Restoration

  • Platform: Microsoft Hyper-V
  • Action: Launch new VM from .vhdx backup.
  • Time Est: < 5 minutes

Step 3: Failback Procedure

  • Timing: Post-business hours
  • Action: Migrate data from VM to repaired hardware.
  • Goal: Return to primary physical infrastructure.

Detailed Proposal Appendix

Both servers are critical to daily business operations and are protected with multiple layers of backup and recovery options.

SCCMAIN

  • Functions: Active Directory, Sage Pastel, and Email Storage
  • Storage: Hardware RAID 1 (IcyDock)

EYETHU1

  • Functions: Point of Sale & Time Manager
  • Storage: Hardware RAID 1 (IcyDock)

Primary Backup Method – Disk2VHD

  • Process: Full system images of SCCMAIN and EYETHU1 are created using Disk2VHD. Backup files (.vhdx) are stored securely on the IT PC’s hard drive. Backups are ready to be mounted and run in Microsoft Hyper-V in the event of hardware failure.
  • Recovery Time (SCCMAIN): 2–3 hours to copy data + 5 minutes to launch in Hyper-V.
  • Recovery Time (EYETHU1): 1 hour to copy data + 5 minutes to launch in Hyper-V.
  • Failover Plan: If a physical server fails, operations are resumed quickly in Hyper-V using the latest .vhdx backup. After store closing, Hyper-V data is migrated back to a physical drive and restored to the original server hardware.

Secondary Backup Method – Veritas Backup

  • Role: Provides an additional safety net.
  • Usage: If the Disk2VHD process fails, Veritas Backup can be used to recover system data and files.
  • Advantage: Ensures business continuity through redundancy.
  • Minimal Downtime: Failover to Hyper-V allows rapid recovery (within minutes once data is copied).
  • Redundancy: Dual backup systems (Disk2VHD + Veritas).
  • Flexibility: Backups are hypervisor-ready and can be restored to both virtual and physical environments.
  • Business Continuity: Ensures core services (Active Directory, Pastel, POS, Time Manager, and Email) remain operational with minimal interruption.

Server Upgrade

  • Current servers are running Windows Server 2012 R2.
  • Planned upgrade path: Migration to Windows Server 2025.
  • Testing is underway using Microsoft’s built-in Server Migration Tools.
  • Benefits: Enhanced performance, stronger security, and longer support lifecycle.
